¡Claro! Aquí tienes una lista de 10 preguntas sobre programación colaborativa utilizando Git, con sus respectivas opciones múltiples:

### 1. ¿Qué comando se utiliza para clonar un repositorio remoto en Git?
a) `git clone <url>`  
b) `git fork <url>`  
c) `git pull <url>`  
d) `git checkout <url>`  

**Respuesta correcta:** a) `git clone <url>`

---

### 2. ¿Cuál de los siguientes comandos se utiliza para agregar cambios al área de preparación (staging area) en Git?
a) `git commit -m "mensaje"`  
b) `git add .`  
c) `git push origin master`  
d) `git pull origin main`  

**Respuesta correcta:** b) `git add .`

---

### 3. Si deseas crear una nueva rama en Git, ¿qué comando usarías?
a) `git switch -c <nombre_de_rama>`  
b) `git branch <nombre_de_rama>`  
c) `git create <nombre_de_rama>`  
d) `git merge <nombre_de_rama>`  

**Respuesta correcta:** b) `git branch <nombre_de_rama>`

---

### 4. ¿Cómo puedes descargar los cambios del repositorio remoto sin realizar una fusión con tu rama local?
a) `git fetch`  
b) `git merge`  
c) `git pull`  
d) `git rebase`  

**Respuesta correcta:** a) `git fetch`

---

### 5. Si estás trabajando en una rama y quieres combinarla con la rama principal (por ejemplo, `main`), ¿qué comando usas?
a) `git merge <nombre_de_rama>`  
b) `git checkout main`  
c) `git rebase <nombre_de_rama>`  
d) `git clone main`  

**Respuesta correcta:** a) `git merge <nombre_de_rama>`

---

### 6. ¿Qué comando se utiliza para eliminar una rama en Git?
a) `git remove <nombre_de_rama>`  
b) `git branch -d <nombre_de_rama>`  
c) `git delete <nombre_de_rama>`  
d) `git clean <nombre_de_rama>`  

**Respuesta correcta:** b) `git branch -d <nombre_de_rama>`

---

### 7. ¿Cuál es el propósito de un "pull request" en Git?
a) Subir cambios a tu repositorio local  
b) Solicitar la fusión de cambios en una rama a otra rama  
c) Descargar los cambios de otro repositorio remoto  
d) Cambiar el historial de commits en una rama  

**Respuesta correcta:** b) Solicitar la fusión de cambios en una rama a otra rama

---

### 8. ¿Qué comando se utiliza para ver los cambios entre tu rama local y la rama remota?
a) `git diff`  
b) `git status`  
c) `git show`  
d) `git pull --rebase`  

**Respuesta correcta:** a) `git diff`

---

### 9. ¿Qué archivo es utilizado por Git para almacenar las configuraciones de tu repositorio local?
a) `.gitignore`  
b) `config.yml`  
c) `.git/config`  
d) `git_settings.json`  

**Respuesta correcta:** c) `.git/config`

---

### 10. Si se hace un "push" a una rama equivocada, ¿qué comando puedes utilizar para deshacerlo?
a) `git push --force`  
b) `git reset --hard HEAD`  
c) `git revert <commit>`  
d) `git undo last push`  

**Respuesta correcta:** c) `git revert <commit>`

---

¡Por supuesto! Aquí tienes 3 preguntas adicionales, diferentes de las anteriores:

### 11. ¿Qué comando de Git te permite ver el historial de commits de un repositorio?
a) `git log`  
b) `git history`  
c) `git commits`  
d) `git show log`  

**Respuesta correcta:** a) `git log`

---

### 12. ¿Qué significa el comando `git pull`?
a) Descargar cambios del repositorio remoto y fusionarlos con la rama actual  
b) Solo descargar los cambios del repositorio remoto sin fusionarlos  
c) Crear una nueva rama en el repositorio remoto  
d) Hacer un commit de los cambios locales en tu repositorio  

**Respuesta correcta:** a) Descargar cambios del repositorio remoto y fusionarlos con la rama actual

---

### 13. ¿Qué archivo se utiliza para indicar a Git qué archivos o directorios deben ser ignorados y no ser versionados?
a) `.git/config`  
b) `.gitignore`  
c) `gitignore.txt`  
d) `.git/settings`  

**Respuesta correcta:** b) `.gitignore`
